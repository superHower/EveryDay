# 易企秀代码管理
```shell

1. 分支管理
        公共分支： master pre dev
        开发分支： feature-dzq-20240201-language
        分支命名规则： 功能—人员—日期—功能
2. 分支间的关系
        开发分支——>测试分支——>预备发布分支——>生产分支
3. 合并分支
    1.开发合并merge公共分支
        要在gitLab上,提交分支合并请求
    2.开发合并merge生产分支
        需要先在master分支拉取pull最新的代码，
        把master的代码合并merge到自己的开发分支,再提交合并请求
4. 提交代码
    1.查看commitlint.config.js文件
    2.实现一个功能，进行一次提交，
    3.代码需要提测的时候，再向公共分支合并代码
    - git checkout dev
    - git pull origin dev
    - git merge feat-hower-20240304-staticPages
    - git push origin dev
    4.提交代码后，部署成功，
      需要回归下功能是否正常，不要影响其他协同开发人员
        feat： 适配吉利车机屏幕
        fix： 增加用户问题
        merge： 解决代码冲突
        chore： 优化依赖
        refactor： 接物游戏逻辑
        style: 登陆弹窗
  5. 获取最新dev分支代码
    - git status ## 确保所有代码都已经commit
    - git checkout feat
    - git fetch origin dev
    - git merge origin dev
```
