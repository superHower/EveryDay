# 1. 概念

## 1.1 原子性
事务会把所有操作作为一个整体，向系统提交撤销操作请求，是不可分割的最小单元，即要么同时成功， 要么同时失败

## 1.2 一致性
事务完成时， 所有状态都保持状态一致

## 1.3. 隔离性
保证事务在不受外部并发操作影响的独立环境下运行

## 1.4 持久性
事务一旦提交或回滚， 对数据的改变就是永久的


## 注意
事务默认自动提交， 一个sql语句是一个事务




```
-- 开启事务
start transaction ;

-- 删除 部门表 的 部门3
delete  from tb_dept where id = 3;

-- 删除 员工表 的 部门为3 的员工
delete  from tb_emp where dept_id = 3;

-- 提交事务
commit;

-- 回滚事务
rollback ;

````
